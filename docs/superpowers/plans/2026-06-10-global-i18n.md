# Global Chinese/English I18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the whole Statistics Learning Studio switch cleanly between Chinese and English, with no mixed-language sentences in the sidebar, main visualizer, right teaching panel, controls, chart labels, metrics, or dynamic interpretations.

**Architecture:** Replace the current iframe DOM text-replacement fallback with native language-aware rendering in each module. The platform owns the language switch and broadcasts the selected language to the active iframe; every module listens for that language and renders from typed copy dictionaries or localized module configs. No partial phrase replacement should remain in production UI.

**Tech Stack:** TypeScript, Vite, Cycle.js, xstream, D3, existing monorepo `apps/*` workspace structure.

---

## Root Cause

The current implementation in `src/main.ts` uses `frameZhTranslations` plus `translateTextValue()` to scan iframe text nodes and replace matching English phrases after the module has already rendered.

That approach causes the exact bug shown in the browser:

```text
A confidence 区间 is an 区间-built estimate...
```

because the translator performs phrase-level replacements inside a longer English sentence. It does not know whether the complete sentence has been localized, so partial terms such as `Confidence Interval`, `interval`, or `method` can be replaced inside an otherwise English sentence.

The correct fix is not adding more dictionary entries. The correct fix is to stop translating rendered DOM text and make each module render either Chinese copy or English copy from the source.

- [ ] **Step 1: Create shared helper**

Create `apps/shared/language.ts`:

```ts
import xs, { type Stream } from "xstream";
import dropRepeats from "xstream/extra/dropRepeats";

export type Language = "zh" | "en";
export const LANGUAGE_STORAGE_KEY = "statistics-platform-language";

export function normalizeLanguage(value: unknown): Language {
  return value === "en" ? "en" : "zh";
}

export function getInitialLanguage(): Language {
  try {
    return normalizeLanguage(window.localStorage.getItem(LANGUAGE_STORAGE_KEY));
  } catch {
    return "zh";
  }
}

export function languageStream(): Stream<Language> {
  return xs
    .create<Language>({
      start(listener) {
        listener.next(getInitialLanguage());

        const onMessage = (event: MessageEvent) => {
          if (event.data?.type !== LANGUAGE_STORAGE_KEY) return;
          const nextLanguage = normalizeLanguage(event.data.language);
          try {
            window.localStorage.setItem(LANGUAGE_STORAGE_KEY, nextLanguage);
          } catch {
            // Ignore storage failures in embedded contexts.
          }
          listener.next(nextLanguage);
        };

        const onStorage = (event: StorageEvent) => {
          if (event.key === LANGUAGE_STORAGE_KEY) {
            listener.next(normalizeLanguage(event.newValue));
          }
        };

        window.addEventListener("message", onMessage);
        window.addEventListener("storage", onStorage);

        this.stop = () => {
          window.removeEventListener("message", onMessage);
          window.removeEventListener("storage", onStorage);
        };
      },
      stop() {}
    })
    .compose(dropRepeats());
}
```

const latinWordPattern = /\b[A-Za-z]{3,}\b/;
const allowedLatin = new Set(["MCMC", "MES", "WALS", "PDF", "PMF", "CDF", "CMF", "GDP", "SSE"]);

function findUnexpectedLatin(text: string): string[] {

    .map((match) => match[0])
    .filter((word) => !allowedLatin.has(word));
